Python入門教學EP3

MAR. 27, 2022

此篇文章通用於大部分作業系統

浮點數(float)

Python 本身就有不少內建的資料型態。前面提過了str和int，我們首先來看看浮點數。
補充:浮點數就是有小數點的數ex.1.2,0.5
浮點數的用法有很多，int能做的，浮點數幾乎也可以做，像是計算:
>>>2.8 + 0.1
2.9
也可以裝入變數:
>>>num = 1.1
>>>num
1.1
實驗:把浮點數放入int()函數中，答案是否與你想的一樣:
>>>int(1.9)
1
欸~怎麼會是1呢，原來int函數是使用無條件捨去法，才會使答案為1。

串列(List)

再來看看串列吧~list顧名思義，是東西的列表。
我們來嘗試建立一個串列:
>>>[]
[]
對!它是空的，那我們要怎麼放資料到串列裡呢?
我們來試試看裝入20以內的質數好了:
prime_number = [2, 3, 5, 7, 11, 13, 17, 19]
這樣就創建完一個list了
我們一樣可以用len()函數來檢測list中的長度:
>>>len(prime_number)

如果想把list內的資料提取出來，可以使用中括號([])告訴list我們要期中第幾個物品。 在電腦裡，事情是從0開始，因此最前面的東西是第0個: >>>prime_number[0]
2
>>> prime_number[1]
3
也可以從後面數回來:
>>>prime_number[-1]
19
>>> prime_number[-2]
17
tips:因為-0跟0相等，所以從後面數時 ，必須從-1開始 串列跟前面提到的int,float,str一樣，它也可以被修改。如果我們要把prime_number list 的2改成1，需要這樣寫:
>>>prime_number[0] = 1
我們在複查一次內容:
>>>prime_number
[1, 3, 5, 7, 11, 13, 17 , 19]
list的第一項確實改變了!

字典(Dictionary 簡稱dict)

字典跟串列很像，都可以存放複數個資料，但是字典是用來儲存資料間對應的關係， 就像是字典內有單字和語義一樣。空的字典長這個樣子:
>>>{}
{}
我們來用一個字典紀錄你的個人資料做為練習。像底下這樣(記得把資料換成你自己的!):
>>>information = {'name': 'E', 'like_num': [1 ,2], 'country':Taiwan, 'favorite_food': 'chocolate'}
字典內會包含多個鍵(key)和值(valse)的組合。在這個範例中， 我們在information這個dict內建立了4樣項目:
。name對應了'E' >資料為str
。like_num對應1, 2 >資料為list,內有兩個int
。country指向'Taiwan' >資料為str
。favorite_food指向'chocolate' >資料為str

如果想檢查dict中某一個key對應到的value，要使用這個語法:
>>>information['name']
'E'
和list差不多，但用dict時，要用key呼叫，而非順序。

如果一個dict沒有某個value，我們依舊詢問Python，它會怎麼處理?
>>> my_information['age']
Traceback (most recent call last):
File "< stdin>", line 1, in < module>
KeyError: 'age'
 呃...又是錯誤!?這次Python給我們新的Error!叫做KeyError， 並告訴我們是哪個key出了問題:age並不存在於前面的dict內。

前面的list可以用len來查看有多少資料，dict當然也可以:
>>>len(information)
4 dict跟list一樣是可變物件。如果要再新增一個key，就要用以下語法:
>>>information['favorite_color'] = 'skyblue'
>>>information
{'name': 'E', 'like_num': [1 ,2], 'country':Taiwan, 'favorite_food': 'chocolate', 'favorite_color': 'skyblue'}

若想刪除裡面的資料，則可以使用del(縮寫自delete):
>>>del information['favorite_color']
>>>information
{'name': 'E', 'like_num': [1 ,2], 'country':Taiwan, 'favorite_food': 'chocolate'}

或者，如果想修改某個key所對應的值，也沒有問題: >>>information['favorite_food'] = 'icecream'
>>>information
{'name': 'E', 'like_num': [1 ,2], 'country':Taiwan, 'favorite_food': 'icecream'}

看完list和dict後，我們可以發現它們有極高的相似度，都可以整理複數資料。 那我們什麼時候要用哪一種呢?
。資料需要順序->list 。資料之間要有對應關係，方便查詢->dict
